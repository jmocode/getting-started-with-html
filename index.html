Introduction To HTML

 Coding

Coding a computer means telling a computer what you want it to do by writing text in a way that it can understand. Also referred to as programming language.

Basics Of The Web

The web is comprised of collections of HTML documents. A web page is a text document written in a language called HTML. Web browsers read these documents, and then interpret and display them. HTML (Hyper Text Markup Language) is the format for the language of the many types of documents on the web. URLs (Uniform Resource Identifier) is the address for documents on the web. HTTP is (Hypertext Transfer Protocol) rules for transferring files (text, graphic images, sound, video, and other multimedia files) on the web. Web Application is any computer program that runs in a web browser.

HTML Basic Lingo

  • Tag — An HTML tag is always contained within angled brackets. Most tags have an opening tag and a closing tag, some tags called void tags do not require a closing tag.

  • Element — An HTML element refers to everything within a set of opening and closing tags.

  • Attribute — This is a property of an HTML element. For example, to set the href attribute of an anchor tag you would write a href="www.url name.com" contained inside of angled brackets.

5 Steps To Thinking Like A Programmer:

  • Procedural Thinking — Procedural thinking involves creating perfectly clear and unambiguous instructions for a computer to follow.

  • Abstract Thinking — Abstract thinking means finding similarity, or as programmers would say, generality amongst seemingly different things.

  • Systems Thinking — Systems thinking happens when you break a big problem down into smaller pieces. Programmers do this when they create a plan (often on paper) for how a program will work. It involves big-picture thinking and decision-making about a problem and how different pieces of a program can work together to solve it.

  • Technological Empathy — "Technological Empathy" comes in many forms. For example, computer empathy is the ability to understand what a computer is, how it works, and what it is good and bad at doing. A computer is a tool, as are the programs we use and the programming languages used to write them. It's almost impossible to program anything substantial without a basic understanding of how these tools work.

  • Debugging — Debugging is a systematic process of relentlessly identifying the cause of a computer program that doesn't work. When a program doesn't work, it's because there is a mistake (also known as a bug) somewhere in the computer code. Since these mistakes are an unavoidable part of programming, every good programmer has a system for fixing them and it usually follows these steps:

    1. Collective evidence (what makes this program fail?)
    2. Generate theories (what may have caused this problem?)
    3. Test those theories (if my theory is correct, how could I find out?)
    4. Fix the problem.

    HTML Examples

    The basic structure for HTML markup is < name > contents < /name > this consists of an opening tag, the contents and a closing tag called an element.

    < b > tag — The b tag renders text BOLD
    < em > tag — The em tag renders text italic
    < br > tag — The break tag creates a line break
    < p > tag — The paragraph tag creates a box around text
    < a > tag — The anchor tag creates a link to another web page
    < img > tag — The image tag adds an image

    Computers Are Stupid

    Programmers often describe computers as "stupid" because computers only understand the instructions that human programmers give to them. This makes them highly sensitive and open to small mistakes (that a human would not make) leading to huge problems.
